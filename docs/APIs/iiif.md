# IIIF Search & Annotations API

**Deliverable:** P3-2.1 (API Documentation — IIIF)
**Date:** March 2026
**Service:** Custom IIIF Search Microservice

---

## Overview

Turath deploys a dedicated Flask microservice implementing the [IIIF Content Search API 1.0](https://iiif.io/api/search/1.0/) and a custom Annotations API for text overlay. The service runs alongside InvenioRDM and is referenced directly from each record's IIIF manifest.

**Production Base URL:** `https://invenio.turath-project.com:5001`

**Infrastructure:** AWS ECS Fargate (`search-service` task), 4 vCPU / 8 GB RAM, backed by an EFS volume containing the HOCR files.

---

```
Browser / Mirador Viewer
        │

  InvenioRDM (port 443)

  │  IIIF Manifest       │──► includes search service URL
  │  (per record)        │
  └──────────────────────┘
        │
        ▼

  ┌─────────────────────────────────────────┐
  │  /search/{record_pid}?q=...             │
  │  /autocomplete/{record_pid}?q=...       │
  │  /annotations/{record_pid}/{page_id}    │
  └─────────────────────────────────────────┘
        │
        ▼
  EFS Volume: /hocr_mount/books/{parent_id}/hocr/
  001.hocr, 002.hocr, ...
```

## Manifest Integration

Each IIIF manifest generated by InvenioRDM embeds the search service URL:

```json
{
  "@context": [
    "http://iiif.io/api/presentation/2/context.json",
    "http://iiif.io/api/search/1/context.json"
  ],
  "service": [
    {
      "@id": "https://invenio.turath-project.com:5001/search/{record_pid}",
      "profile": "http://iiif.io/api/search/1/search",
      "label": "Search within this document",
      "service": {
        "@id": "https://invenio.turath-project.com:5001/autocomplete/{record_pid}",
        "profile": "http://iiif.io/api/search/1/autocomplete"
      }
    }
  ]
}
```

## Endpoints

### 1. Search within a Record

Searches for a query across all HOCR pages of a record and returns an AnnotationList with bounding-box hits, compatible with Mirador's built-in search panel.

**Endpoint:** `GET /search/{record_pid}`

**Parameters:**
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `q` | string | Yes | Search query (Arabic or English phrase) |

**Example:**
```bash
curl "https://invenio.turath-project.com:5001/search/abc12-def34?q=تاريخ"
```

**Response (IIIF Content Search API 1.0):**
```json
{
  "@context": "http://iiif.io/api/search/1/context.json",
  "@id": "https://invenio.turath-project.com:5001/search/abc12-def34?q=تاريخ",
  "@type": "sc:AnnotationList",
  "resources": [
    {
      "@id": "https://invenio.turath-project.com:5001/annotations/abc12-def34/3/0",
      "@type": "oa:Annotation",
      "motivation": "sc:painting",
      "resource": {
        "@type": "cnt:ContentAsText",
        "chars": "تاريخ"
      },
      "on": "https://invenio.turath-project.com/records/abc12-def34/canvas/p003#xywh=120,450,85,30"
    }
  ],
  "hits": [
    {
      "@type": "search:Hit",
      "annotations": ["https://invenio.turath-project.com:5001/annotations/abc12-def34/3/0"],
      "match": "تاريخ",
      "before": "... كتاب في ",
      "after": " نجد ...",
      "on": "https://invenio.turath-project.com/records/abc12-def34/canvas/p003"
    }
  ]
}
```

**Notes:**
- Search is **parallel** across all HOCR pages (ProcessPoolExecutor, 4 workers)
- Supports **phrase matching** and **case-insensitive** matching
- Returns union bounding boxes for multi-word phrases
- Returns empty `resources`/`hits` arrays (not 404) when no HOCR files exist

### 2. Autocomplete

Returns term suggestions based on a query prefix, used by Mirador's search panel for autocomplete.

**Endpoint:** `GET /autocomplete/{record_pid}`

**Parameters:**
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `q` | string | Yes | Partial word prefix to complete |

**Example:**
```bash
curl "https://invenio.turath-project.com:5001/autocomplete/abc12-def34?q=تار"
```

**Response (IIIF Content Search API 1.0 — TermList):**
```json
{
  "@context": "http://iiif.io/api/search/1/context.json",
  "@id": "https://invenio.turath-project.com:5001/autocomplete/abc12-def34?q=تار",
  "@type": "search:TermList",
  "terms": [
    {
      "match": "تاريخ",
      "url": "https://invenio.turath-project.com:5001/search/abc12-def34?q=تاريخ",
      "count": 1
    }
  ]
}
```

**Notes:**
- Scans the first 5 HOCR pages for prefix matches (performance optimisation)
- Returns up to 10 suggestions per request

---

### 3. Annotations (Text Overlay)

Returns a full IIIF AnnotationList for a specific page — all words with their bounding boxes. This powers the Mirador text overlay plugin, which renders every recognised word as a selectable, hoverable layer over the scanned image.

**Endpoint:** `GET /annotations/{record_pid}/{page_id}`

**Parameters:**
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `record_pid` | string | Yes | InvenioRDM record persistent identifier |
| `page_id` | string | Yes | Page identifier in `pXXX` format (e.g., `p001`, `p042`) |

**Example:**
```bash
curl "https://invenio.turath-project.com:5001/annotations/abc12-def34/p001"
```

**Response:**
```json
{
  "@context": "http://iiif.io/api/presentation/2/context.json",
  "@id": "https://invenio.turath-project.com:5001/annotations/abc12-def34/p001",
  "@type": "sc:AnnotationList",
  "resources": [
    {
      "@id": "https://invenio.turath-project.com:5001/annotations/abc12-def34/p001/0",
      "@type": "oa:Annotation",
      "motivation": "sc:painting",
      "resource": {
        "@type": "cnt:ContentAsText",
        "chars": "بسم",
        "format": "text/plain"
      },
      "on": "https://invenio.turath-project.com/records/abc12-def34/canvas/p001#xywh=100,200,60,25"
    }
  ]
}
```

**Bounding Box Format:** `x,y,width,height` in pixels (derived from HOCR `bbox` attribute `x1 y1 x2 y2`)

**Error Responses:**
- `400 Bad Request` — `page_id` not in `pXXX` format
- `404 Not Found` — HOCR file for the specified page does not exist

## HOCR File Structure

The service expects HOCR files mounted at:

```
/hocr_mount/books/{parent_id}/hocr/
├── 001.hocr
├── 002.hocr
├── 003.hocr
└── ...
```

- `parent_id` is resolved from `record_pid` by calling the InvenioRDM API (`/api/records/{pid}`)
- Pages are sorted lexicographically (`001.hocr` = page 1, `042.hocr` = page 42)
- HOCR files follow the standard HTML-based OCR format with `ocrx_word` spans and `bbox` attributes

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `HOCR_BASE_DIR` | `/hocr_mount/books` | Filesystem path to HOCR storage |
| `IIIF_SERVER_BASE_URL` | `https://127.0.0.1:5000` | InvenioRDM base URL |
| `APP_BASE_URL` | (same as above) | URL for server-side API calls to InvenioRDM |
| `CANVAS_BASE_URL` | (same as above) | Base URL for browser-facing canvas URIs in manifests |
| `SEARCH_SERVICE_BASE_URL` | `https://127.0.0.1:5001` | Public URL of this service (used in responses) |
| `MAX_WORKERS` | `4` | Number of parallel workers for HOCR page search |

## Deployment

The service is containerised and deployed on AWS ECS Fargate:

- **Image:** `ghcr.io/alabarazi/turath-search-service:develop`
- **Port:** `5001`
- **Command:** `gunicorn --bind 0.0.0.0:5001 --workers 4 app:app`
- **EFS Mount:** HOCR files mounted from shared EFS at `/hocr_mount`
- **ALB Route:** Requests to `:5001/search/*`, `:5001/autocomplete/*`, `:5001/annotations/*` are routed to this service
